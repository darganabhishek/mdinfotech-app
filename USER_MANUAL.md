# M.D. INFOTECH - User Manual

Welcome to the M.D. INFOTECH Institute Management Application. This guide outlines the features and functionalities available to each user role.

---

## 🔐 Roles Overview

### 1. Admin / Superadmin
Admins have full access to the system. They manage the institute's core data, finances, and user accounts.
- **Core Management**: Students, Courses, Batches, Admissions, and Faculty.
- **Finance**: Fee Management, Payments, Receipts, and Payroll.
- **System**: Role Management, Audit Logs, Security Alerts, and Settings.
- **Analytics**: Course Analytics and detailed Reports.

### 2. Teacher / Faculty
Teachers focus on academics and attendance.
- **Academic Tools**: Timetable, E-learning Hub, and Resources.
- **Attendance**:
  - **Start QR Session**: Generate a unique QR code for students to scan in class.
  - **Scan QR Code**: Manually scan student QR codes if needed.
  - **Faculty Clock**: Record teaching hours/sessions.
- **Classes**: View assigned Batches and Student lists.

### 3. Student
Students have a dedicated portal for their academic journey.
- **Dashboard**: Overview of enrolled courses, fees paid, and attendance.
- **Academic**: Access E-learning Hub for study materials and Timetable for class schedules.
- **Inquiry**: Track referrals and enquire about new courses.
- **Registration**: Public registration form (accessible via QR code in the Admissions office).

### 4. Staff
Staff members typically handle administrative support and admissions.
- **Admissions**: Process new student applications and enquiries.
- **Students**: Manage student records and communication.
- **Attendance**: Assist with attendance tracking and hub management.
- **Finance**: View and process payments and receipts.

---

## 🚀 Key Features & How to Use Them

### 📁 Admissions & Registration
- **Student Registration**: New students fill out the registration form (Name, 10-digit Phone, 12-digit Aadhaar, Course, etc.).
- **Admin Approval**: Admins review "Pending" admissions in the **Admissions** tab to assign batches and approve them.
- **QR Code**: Admins can click **"View QR"** in the Admissions tab to show the registration link to walk-in students.

### 📑 Attendance Hub
- **QR Sessions**: Teachers start a session. Students scan the code to mark attendance.
- **Security**: The system tracks duplicate scans or unauthorized location attempts (Security Alerts).

### 💰 Fee Management
- **Payments**: Search students, record payments (Full or Installments), and view history.
- **Receipts**: Automatically generates a professional receipt (MDI-XXXX) that can be downloaded or printed.
- **Bulk Import/Export**: Use the **Import CSV** feature in Students/Admissions to batch-upload data using the provided templates.

---

## 🛠️ System Maintenance
- **Role Management**: Define custom permissions for Staff and Teachers.
- **Audit Logs**: Track every major action taken within the app for security.
- **Settings**: Update Institute name, logo, and general preferences.
